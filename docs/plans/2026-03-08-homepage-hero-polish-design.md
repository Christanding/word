# Homepage Hero Polish Design

## Goal

Refine the homepage hero into a stronger editorial front-cover composition with a more memorable presence, using layout, spacing, hierarchy, and monochrome iconography instead of decorative effects.

## Confirmed Direction

- Use the `Cover / Editorial Front Page` approach
- Keep the current fullscreen homepage shell
- Keep all existing functionality unchanged
- Keep the current font system unchanged
- Preserve the embedded header language switcher
- Replace the colorful feature icons with more restrained monochrome editorial symbols

## Design Intent

The homepage hero should feel more like the opening spread of a literary periodical and less like a polished SaaS dashboard.

The memory point should come from:

- stronger scale contrast
- more deliberate whitespace
- tighter cover-like composition
- quieter, more coherent icon language

## Structure

### 1. Kicker / Editorial Note

Keep a small page-level editorial note above the headline, but make it feel more intentional and less like a utility label.

It should function like a cover-line, not a dashboard micro-label.

### 2. Headline

The main title should become the visual anchor.

- larger scale
- lighter weight than a typical product hero
- stronger vertical presence
- more controlled width

The title should feel like a publication masthead or cover headline.

### 3. Subtitle

The subtitle should narrow slightly and behave like a cover subheading.

- more measured width
- less generic product-marketing energy
- stronger reading rhythm

### 4. Transition Into Feature Grid

The gap between hero and feature entry cards should become more intentional.

The cards should feel like a table of contents or section index, not a generic feature grid placed under a hero.

## Icon Strategy

The current feature icons should be replaced with a restrained monochrome set.

Target feel:

- archival
- editorial
- symbol-like
- quiet, not playful

Avoid:

- colorful emoji-like icons
- soft app-store style stickers
- toy-like gradients

Preferred icon behavior:

- single-color or near-single-color
- slightly symbolic / emblematic
- visually consistent across all four feature cards

## Visual Rules

- no new font changes
- no new global token changes
- no extra animation system
- no business logic changes
- no homepage information architecture changes

Only adjust:

- spacing
- hierarchy
- hero scale
- icon treatment
- cover-like rhythm between hero and feature grid

## Success Criteria

- homepage hero feels more like a cover page than a product banner
- title and subtitle have stronger editorial presence
- feature icons feel visually consistent and more refined
- homepage remains fast to understand and use
- no functionality changes
